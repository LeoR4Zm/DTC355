CSS Measuring Units

This Code Log explains how CSS measures things (text, spacing, and boxes). Start by learning the difference between absolute and relative length units, then dive into three common groups used in real projects.

Absolute vs. Relative Units

Absolute units represent fixed physical sizes. They do not depend on the element or viewport. Examples: px (CSS pixel), pt (points, 1/72 of an inch), in (inches), cm, mm, pc (picas), Q. In practice, browsers map these to screen pixels using a standard conversion (e.g., 1in = 96px, 1pt = 1/72in). Absolute units are predictable for print and icons, but they don’t adapt to user settings.

Relative units scale based on context—font sizes, parent dimensions, or the viewport. Examples: em, rem, %, vw, vh, vmin, vmax, ch, ex, cap, lh, rlh. Relative units improve accessibility and responsiveness because they grow/shrink with user preferences or screen size.

Rule of thumb: Use relative units for text and layout so your page adapts; reserve absolute units for pixel-precise artwork, borders, or print CSS.

1) px vs pt (absolute)

What they are

  • px: CSS pixel—device-independent “reference pixel”. Stable for on-screen UI.
  • pt: Points for print (1pt = 1/72 inch). Better for print stylesheets than screens.

When to use

  • px: borders, hairline icons, shadows, fine-tuned spacing.
  • pt: print @media rules where physical sizing matters.
/* Pixel-perfect border and icon example */
.button { border: 1px solid #1f2937; box-shadow: 0 2px 8px rgba(0,0,0,.15); }
.icon   { width: 24px; height: 24px; }

/* Print stylesheet sample */
@media print {
  body { font-size: 11pt; margin: 12mm; }
}
width: 240px
Always 240 CSS pixels wide

2) em vs rem (relative to font size)

What they are

  • 1em equals the computed font-size of the current element (or its parent when used on font-size).
  • 1rem equals the font-size of the <html> element (root). Stable across nesting.

When to use

  • rem for global typography scale (headings, spacing system).
  • em for components that should scale with their own font-size (buttons, badges).
html { font-size: 16px; }
:root { /* optional: a type scale */ }
h1 { font-size: 2.5rem; margin-bottom: 1rem; }
.card { padding: 1rem; border-radius: .75rem; }
.button { font-size: 0.875rem; padding: 0.75em 1.25em; /* scales with button text */ }
width: 12em
Changes if element font-size changes
width: 12rem
Always 12 × root font-size

3) vw vs vh (relative to viewport)

What they are

  • 1vw = 1% of the viewport width.
  • 1vh = 1% of the viewport height.

When to use

  • Hero sections that fill the screen (min-height: 100vh).
  • Fluid typography/boxes that adapt to device width.
/* Full-screen hero */
.hero { min-height: 100vh; display: grid; place-items: center; }

/* Fluid box widths */
.sidebar { width: min(28vw, 420px); }
width: 30vw
Grows/shrinks with viewport width
width: 20vh
Tied to viewport height

Bonus) % (relative to container)

% resolves against the containing block. For width/height it’s usually the content box of the nearest positioned ancestor; for font-size it inherits from the parent’s font-size; for transforms and some properties it can behave differently. Great for fluid layouts within a fixed grid.

.content { width: 60%; margin-inline: auto; }
.avatar  { width: 25%; /* one quarter of its container */ }
width: 60%

Key Takeaways

  • Prefer relative units (rem, em, %, vw/vh) for accessibility and responsive design.
  • Use absolute units (px, pt, in, etc.) sparingly, mainly for print and pixel-precise details.
  • Build a type scale with rem, let components scale locally with em, and make big sections fluid with vw/vh.

/* Example: fluid type using clamp() */
:root { font-size: 16px; }
h1 { font-size: clamp(1.75rem, 2.5vw + 1rem, 2.5rem); }

/* Spacing scale anchored to rem */
:root { --space-1: 0.25rem; --space-2: 0.5rem; --space-3: 1rem; --space-4: 1.5rem; }
.section { padding: var(--space-4) var(--space-3); }

/* Component that scales with its own font size */
.badge { font-size: .875rem; padding: .3em .6em; border-radius: 999px; }
